--- title: "Alert Popup" layout: xtag root: "../" description: "An alert popup dialog is a small window which demands user's input related to the current activity. It disables all current activities and only disappears after the user confirms an action or dismisses the dialog." use_it: "When the current activity absolutely requires user input, usually validation (i.e. delete all contacts?) or authorization (i.e. enable GPS?)." ---

Do's

  • - action oriented (i.e. delete all contacts?)
  • - short questions or statements
  • - as a rule of thumb, place the positive button on the right, and the negative button on the left
  • - use descriptive, free standing labels such as "Connect"

Dont's

  • - don't use negation in your question or statement
  • - don't use it when the action is trivial and reversible. Use a toast instead.
  • - don't use it when you want the user to make a trivial choices. Consider a select list instead.
